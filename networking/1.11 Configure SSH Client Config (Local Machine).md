# Task 1.11: Configure SSH Client Config (Local Machine)

**Objective**: Set up SSH config file for easy FQDN-based access (both themed and descriptive names).

**Steps**:
1. On your local machine (laptop/desktop), create/edit SSH config:
   ```bash
   mkdir -p ~/.ssh
   touch ~/.ssh/config
   chmod 600 ~/.ssh/config
   nano ~/.ssh/config
   ```

2. Add the following entries (both themed and descriptive names):
   ```ssh-config
   # OPNsense Router - Themed name (shire)
   Host shire shire.cuffney.com
       HostName shire.cuffney.com
       User root
       Port 22
       IdentityFile ~/.ssh/id_ed25519

   # OPNsense Router - Descriptive name (opnsense)
   Host opnsense opnsense.cuffney.com
       HostName opnsense.cuffney.com
       User root
       Port 22
       IdentityFile ~/.ssh/id_ed25519

   # Proxmox Host - Themed name (numenor)
   Host numenor numenor.cuffney.com
       HostName numenor.cuffney.com
       User root
       Port 22
       IdentityFile ~/.ssh/id_ed25519

   # Proxmox Host - Descriptive name (proxmox)
   Host proxmox proxmox.cuffney.com
       HostName proxmox.cuffney.com
       User root
       Port 22
       IdentityFile ~/.ssh/id_ed25519
   ```

3. Adjust `IdentityFile` to match your SSH key path if different

**Test**:
```bash
# Test DNS resolution first (all variations)
nslookup shire.cuffney.com
nslookup opnsense.cuffney.com
nslookup numenor.cuffney.com
nslookup proxmox.cuffney.com

# Test SSH connection using themed FQDNs
ssh jcuffney@shire.cuffney.com
# Should connect to OPNsense (if SSH is enabled)

ssh jcuffney@numenor.cuffney.com
# Should connect to Proxmox

# Test SSH connection using descriptive FQDNs
ssh jcuffney@opnsense.cuffney.com
# Should connect to OPNsense

ssh jcuffney@proxmox.cuffney.com
# Should connect to Proxmox

# Test SSH using short aliases
ssh shire
ssh opnsense
ssh numenor
ssh proxmox
```

**Expected Result**: Can SSH using either themed or descriptive FQDNs: `ssh jcuffney@shire.cuffney.com` or `ssh jcuffney@opnsense.cuffney.com`, etc.
