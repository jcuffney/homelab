# Task 1.10: Add DNS Override for OPNsense (FQDN)

**Objective**: Create DNS entries so both `opnsense` and `shire` (and their FQDNs) resolve to `192.168.1.1`.

**Steps**:
1. In OPNsense: **Services → Unbound DNS → Overrides**
2. Create first override (themed name):
   - Click **Add** button
   - Configure:
     - **Host**: `shire`
     - **Domain**: `cuffney.com`
     - **Type**: `A`
     - **Value**: `192.168.1.1`
     - **Description**: `OPNsense router (shire - themed)`
   - Click **Save**
3. Create second override (descriptive name):
   - Click **Add** button
   - Configure:
     - **Host**: `opnsense`
     - **Domain**: `cuffney.com`
     - **Type**: `A`
     - **Value**: `192.168.1.1`
     - **Description**: `OPNsense router (opnsense - descriptive)`
   - Click **Save**
4. Create third override (short themed name, no domain):
   - Click **Add** button
   - Configure:
     - **Host**: `shire`
     - **Domain**: (leave empty)
     - **Type**: `A`
     - **Value**: `192.168.1.1`
     - **Description**: `OPNsense router (shire - short)`
   - Click **Save**
5. Create fourth override (short descriptive name, no domain):
   - Click **Add** button
   - Configure:
     - **Host**: `opnsense`
     - **Domain**: (leave empty)
     - **Type**: `A`
     - **Value**: `192.168.1.1`
     - **Description**: `OPNsense router (opnsense - short)`
   - Click **Save**
6. Click **Apply Changes**

**Test**:
```bash
# Test DNS resolution (all variations should work)
nslookup shire
# Should resolve to: 192.168.1.1

nslookup opnsense
# Should resolve to: 192.168.1.1

nslookup shire.cuffney.com
# Should resolve to: 192.168.1.1

nslookup opnsense.cuffney.com
# Should resolve to: 192.168.1.1

dig @192.168.1.1 shire.cuffney.com
# Should return: shire.cuffney.com.  IN A 192.168.1.1

dig @192.168.1.1 opnsense.cuffney.com
# Should return: opnsense.cuffney.com.  IN A 192.168.1.1
```

**Expected Result**: Both `shire.cuffney.com` and `opnsense.cuffney.com` (and short names) resolve to `192.168.1.1`.
