# Task 1.9: Add DNS Override for Proxmox (FQDN)

**Objective**: Create DNS entries so `numenor` (and its FQDN) resolves to `192.168.1.10`.

## Hosts vs Aliases

In OPNsense Unbound DNS Overrides:
- **Hosts**: Primary DNS entries (A records) - the main hostname
- **Aliases**: Secondary names that point to a host - alternative names for the same IP

**For this task**: Use **Hosts** to create the primary DNS entries. Aliases are optional if you want additional names.

**Steps**:
1. In OPNsense: **Services → Unbound DNS → Overrides**
2. Navigate to the **Hosts** section (not Aliases)
3. Create first host override (FQDN with domain):
   - Click **Add** button
   - Configure:
     - **Host**: `numenor`
     - **Domain**: `cuffney.com`
     - **Type**: `A`
     - **Value**: `192.168.1.10`
     - **Description**: `Proxmox host (numenor)`
   - Click **Save**
4. Create second host override (short name, no domain):
   - Click **Add** button
   - Configure:
     - **Host**: `numenor`
     - **Domain**: (leave empty)
     - **Type**: `A`
     - **Value**: `192.168.1.10`
     - **Description**: `Proxmox host (numenor - short)`
   - Click **Save**
5. Click **Apply Changes**

**Note**: If you want `proxmox` to also resolve (descriptive name), you can either:
- Create additional Host entries for `proxmox` (same as above but with `proxmox` as hostname), OR
- Create Alias entries pointing to `numenor` (in the Aliases section)

**Test**:
```bash
# Test DNS resolution
nslookup numenor
# Should resolve to: 192.168.1.10

nslookup numenor.cuffney.com
# Should resolve to: 192.168.1.10

dig @192.168.1.1 numenor.cuffney.com
# Should return: numenor.cuffney.com.  IN A 192.168.1.10

dig @192.168.1.1 numenor
# Should return: numenor.  IN A 192.168.1.10
```

**Expected Result**: `numenor.cuffney.com` and `numenor` (short name) resolve to `192.168.1.10`.
